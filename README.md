# Product REST API

Simple REST API built with Spring Boot and PostgreSQL.

## Features
- CRUD operations for products
- PostgreSQL persistence with JPA/Hibernate
- Validation for input data
- REST endpoints
- Optional database management using DBeaver

## Endpoints

| Method | Endpoint       | Description                |
|--------|----------------|----------------------------|
| GET    | /products      | Get all products           |
| GET    | /products/{id} | Get product by ID          |
| POST   | /products      | Create a new product       |
| PUT    | /products/{id} | Update an existing product |
| DELETE | /products/{id} | Delete a product           |

## Tech stack
- Java 17+ (or your version)
- Spring Boot
- PostgreSQL
- JPA / Hibernate
- Maven (for dependency management)
- DBeaver (optional, for visual DB management)

## Setup Instructions

1. **PostgreSQL**
    - Install PostgreSQL.
    - Create a database, e.g., `ProductDB`.
    - Note your username and password.

2. **Application configuration**
    - Edit `src/main/resources/application.properties`:

```properties
spring.datasource.url=jdbc:postgresql://localhost:5432/ProductDB
spring.datasource.username=your_username
spring.datasource.password=your_password

spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
```

3. **Run the application**

    Use your IDE (IntelliJ/Eclipse) or run via Maven:

    - mvn spring-boot:run
   

4. **Testing with Postman**

    - Send requests to http://localhost:8080/products.

    - Example JSON for creating a product:
```properties
    {
    "name": "Laptop",
    "price": 5000
    }
```

5. **Optional: DBeaver**

    Connect to ProductDB to view tables and data.


# **Notes** 
    1. IDs are automatically generated by the database.
    2. Deleted IDs are not reused automatically; new products get the next available ID.
    3. Validation errors will return HTTP 400 with descriptive messages.
